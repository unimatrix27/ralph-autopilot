# Standardize on zod v4 across ralph-autopilot

The project uses zod **v4** for all schema work (config, the `escalate` tool's
input contract, JSON hand-off contracts). The driver is hard: the executor is the
Claude Agent SDK (ADR-0008), which peer-requires `zod ^4`, and the in-process
`escalate` tool is defined via the SDK's `tool()` helper, which expects a v4
schema. Mixing zod versions across that boundary fails `instanceof` checks at
runtime.

## Consequences

- The foundation initially pinned zod v3 (an arbitrary choice by the bootstrap
  agent); it was migrated to v4. The only non-mechanical change: object-level
  optional blocks use `.prefault({})` instead of `.default({})`, because v4's
  `.default()` validates against the *output* type and skips nested field
  defaults, whereas `.prefault()` applies the value as input and parses it.
- This decision is the canonical example behind the [design-authority
  rule](0011-design-authority-rule.md): the version conflict is resolved toward
  the SDK, not by routing around it.
